# OpenElin Setup Screen Plan (2026-02-23)

This plan is intentionally split into small, self-contained implementation steps.
Each step has a clear scope, expected result, and review checklist so we can implement and validate incrementally.

## Scope (MVP)

- Add a new device subpage named **Setup**.
- Show current configured cog count (same data source as Cogs).
- Include the two explanatory text blocks.
- Add Setup mode with 3 steps:
  1. cog amount select (5..14, default 12),
  2. smallest cog offset,
  3. largest cog offset.
- Render preview table progressively:
  - after step 1: scaffold only,
  - after step 2: smallest value shown,
  - after step 3: full interpolated values shown.
- Enable **Write to NXS** only when interpolation is ready.
- Write using `BLE_CMD_SET_REAR_COG_INFO (0x001D)` with interpolated offsets + teeth placeholders (`0`).
- Demo mode mocks must support this flow.
- Add e2e coverage and Chrome DevTools MCP demo-mode validation.

## Step 5 — Smallest/Largest Offset Inputs + Interpolation Display

### Goal

Implement Steps 2 and 3 with progressive table filling and interpolation.

### Changes

- Add Step 2 input: smallest cog offset.
- Add Step 3 input: largest cog offset.
- Validation rules:
  - both numeric,
  - cog count >= 2,
  - largest > smallest.
- Preview behavior:
  - after valid smallest: first cog offset cell filled,
  - after valid largest: interpolate full offset array and fill all rows.
- Use deterministic interpolation formula over `count` cogs.

### Expected Result

### Review Checklist

## Step 6 — Write to NXS Action (0x001D)

  - build teeth array of same length with `0` placeholders,
  - call existing write path using `SET_REAR_COG_INFO (0x001D)`,

- Reuse existing command/store flow; avoid protocol format duplication.

### Expected Result

Successful write updates cog configuration and reflected cog count/state.

### Review Checklist

- Button is disabled before interpolation is complete.
- Button enables exactly when step 3 is valid.
- Write call succeeds in demo mode.
- Post-write read refreshes displayed count/data.

### Output To Review
