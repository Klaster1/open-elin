# OpenElin Setup Screen Plan (2026-02-23)

This plan is intentionally split into small, self-contained implementation steps.
Each step has a clear scope, expected result, and review checklist so we can implement and validate incrementally.

## Scope (MVP)

- Add a new device subpage named **Setup**.
- Show current configured cog count (same data source as Cogs).
- Include the two explanatory text blocks.
- Add Setup mode with 3 steps:
  1. cog amount select (5..14, default 12),
  2. smallest cog offset,
  3. largest cog offset.
- Add Step 4 finish guidance after successful write:
  1. link to Cogs for per-cog performance tuning,
  2. save current setup as profile (same dialog flow as Cogs) and redirect to Cogs after save.
- In Step 1, add optional indexed cassette tooth-count inputs, defaulting to `0`.
- Render preview table progressively:
  - after step 1: scaffold only,
  - after step 2: smallest value shown,
  - after step 3: full interpolated values shown.
- Enable **Write to NXS** only when interpolation is ready.
- Write using `BLE_CMD_SET_REAR_COG_INFO (0x001D)` with interpolated offsets + teeth placeholders (`0`).
- Demo mode mocks must support this flow.
- Add e2e coverage and Chrome DevTools MCP demo-mode validation.
